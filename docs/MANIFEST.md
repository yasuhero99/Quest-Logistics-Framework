# QLF Manifest

`manifest.json` is the logistics map generated by `resolve-text --manifest`.

It tells QLF where each translation key came from and where it should go during inject.

## Current manifest version

```json
{
  "qlf_manifest_version": 3,
  "source_system": "adapter"
}
```

## Top-level fields

Common fields:

```text
qlf_manifest_version
source_system
pack_root
source_locale
mode
sources
keys
```

## Sources

`sources` records the source files involved in extraction.

Example:

```json
{
  "id": 0,
  "source": "ftbquests",
  "adapter": "ftbquests",
  "source_file": "C:/.../config/ftbquests/quests/lang/en_us.snbt",
  "relative_path": "config/ftbquests/quests/lang/en_us.snbt",
  "format": "snbt",
  "source_locale": "en_us",
  "mode": "native-ftbquests-lang"
}
```

## Keys

`keys` records per-key metadata.

Example:

```json
{
  "quest.123.title": {
    "source": "ftbquests",
    "adapter": "ftbquests",
    "source_id": 0,
    "value_type": "string",
    "line_count": 1
  }
}
```

## Why value_type matters

FTB Quests descriptions can be strings or lists. Earlier versions could falsely report string/list mismatches.

Manifest v2+ records:

```text
value_type
line_count
```

so validate and inject can preserve the expected structure.

## Why source/adapter matters

Before v1.9, FTB Quests was the only source and could be handled as a special case.

After v1.9, QLF supports an adapter model.

This means each key must know which adapter owns it:

```json
"adapter": "ftbquests"
```

Future keys may belong to:

```text
patchouli
quest_kubejs
```

and inject will route them through the correct adapter.
